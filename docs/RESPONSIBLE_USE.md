# Responsible Use

## Intended Use

This project is intended for local development, experimentation, and productivity workflows.

## Safety Notes

- Validate generated code before production use.
- Do not treat model output as legal, medical, or financial advice.
- Review generated content for policy and compliance constraints relevant to your environment.

## Privacy Notes

- Offline mode keeps traffic local by default.
- Hybrid mode may send selected requests to cloud providers when enabled.
- Avoid placing secrets in prompts unless your policy allows it.

## Operational Guardrails

- Keep API keys in secure local stores.
- Rotate credentials when exposure is suspected.
- Log only what is needed for debugging and avoid storing sensitive data.
